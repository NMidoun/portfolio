Description générale

L'utilisateur peut naviguer en tant que visiteur ou en tant qu'utilisateur authentifié.

Si il est authentifié, son panier sera sauvegardé dans Firestore, sinon dans le localstorage.

Les vues du panier sont différentes selon s'il est authentifié ou non.

Si l'url ou une fonctionnalité n'existe pas, une vue spécifique sera affichée.

Description technique

Le store, vide au départ, est traité par Redux. Les méthodes de l'api Firestore et de Firebase Authentication sont transmises à l'app avec l'api context de React.

Au montage du composant App, les produits sont récupérés sur Firestore.

Si l'utilisateur est authentifié, son panier est également récupéré sur Firestore, sinon dans le localStorage.

Dans les deux cas, le panier, s'il existe, est transmis au store Redux.

A chaque ajout d'un article ou à chaque modification depuis le panier, selon si l'utilisateur est authentifié ou non, le panier sera mis à jour soit dans le localStorage soit dans Firestore.

Un utilisateur peut s'inscrire ou s'identifier, les formulaires sont gérer par la bibliothèque Formik.

Lorsqu'il soumet le formulaire d'inscription, si l'email existe dejà, un message d'erreur apparaît.

Si il essaye de s'authentifier et que l'email n'existe pas ou que le mot de passe est incorrect, un message d'erreur spécifique apparaît.
